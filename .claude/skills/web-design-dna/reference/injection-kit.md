# Injection Kit — the output contract

The whole point: take extracted DNA and make it *droppable* into the user's own builds.
Generated by `scripts/lib/report.mjs`; this documents the contract and how to apply it.

## What the kit contains
1. **Portable CSS variables** — a framework-agnostic `:root { … }` of fonts, named colors (roles resolved), the type scale, the spacing scale, the signature easing, and durations. Outlives any one stack.
2. **Tailwind v4 `@theme` block** — the same tokens in this repo's CSS-first format (no `tailwind.config.js`). Paste into `src/index.css`.
3. **JSON tokens** — machine-readable for tooling or a design-token pipeline.
4. **Technique recipes** — copy-paste React/Vite snippets for each detected signature move (Lenis smooth-scroll, GSAP ScrollTrigger reveal, SplitText reveal, R3F shader hero, page transitions, fluid clamp scale). Only recipes relevant to the extracted stack are emitted.
5. **Brand-conflict flags** — where the DNA collides with the target project's rules.

## How to apply it (React / Vite / Tailwind v4)
1. **Tokens first.** Paste the `@theme` block into `src/index.css`; reconcile names with existing tokens (don't clobber the brand palette — add a namespaced scale or map onto existing roles).
2. **Type scale** → your `typography.ts` (or `@theme` `--text-*`). Keep fluid `clamp()` values; retune min/max to your grid.
3. **Motion** → adopt the signature easing as a shared token; gate every animation behind reduced-motion (`isMotionDisabled()`).
4. **Techniques** → drop recipes in as hooks/components, wire to real content, then tune. Treat them as scaffolds, not finished features.
5. **Verify for real.** Run the app and watch it — green tests are not proof a visual/motion feature works. For anything touching WebGL/scroll/streaming, exercise the live path.

## Brand profile schema (`--brand path/to/brand.json`)
```json
{
  "name": "My Project",
  "fonts": {
    "allowed": ["SF Pro Display", "-apple-system", "system-ui"],
    "forbiddenSources": ["Google Fonts"],
    "note": "SF Pro Display, weight 200. Never use Google Fonts."
  },
  "palette": { "brand-dark": "#0A0F1C", "brand-gold": "#C5A572" },
  "noEmojis": true,
  "requireCspUpdateForCdns": true,
  "motionMustRespectReducedMotion": true
}
```
- Default (no `--brand`) = the built-in **Altivum / thechrisgrey** profile (`DEFAULT_BRAND` in `report.mjs`).
- `--brand none` = pure faithful extraction, no flagging.
- Inside another repo, prefer that repo's real design system (its `@theme` / CLAUDE.md) over the default.

## Conflict severities
- **high** — violates a hard brand rule (e.g. forbidden font source). Must resolve before shipping.
- **medium** — foreign typeface, or a CDN dependency needing self-hosting / a CSP allowlist update (`amplify.yml`).
- **low** — reminders (gate heavy motion behind reduced-motion).
- **info** — standing rules (no emojis → use Material Icons / inline SVG).

## Philosophy: extract faithfully, inject deliberately
Capture the real DNA at full fidelity — never water it down at extraction time. Apply taste
at *injection* time: keep the *scale, rhythm, easing, and composition* (the transferable
craft), and re-map surface specifics (exact typeface, exact palette) onto the target brand.
The signature **moves** transfer; the brand **skin** stays the user's.
